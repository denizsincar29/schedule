# Введение
В Северном Арктическом Федеральном Университете (САФУ) доступ к информационным ресурсам осуществляется через две платформы: SAKAI и Modeus. Через последнюю студенты и преподаватели могут просматривать расписание занятий, оценки, посещаемость и другую информацию. К Modeus подключён не только САФУ, но и ТюмГУ и возможно другие ВУЗы.
В рамках данного исследования была разработана программа для просмотра расписания занятий в Северном (Арктическом) федеральном университете.
# 1 Определение требований к разрабатываемому приложению (краткое техническое задание) 
Программа разрабатывается на Python3 с использованием библиотек WXPython и HTTPX.
Программа в первую очередь ориентирована на незрячих пользователей, поэтому для графического интерфейса используется библиотека [WXPython](https://wxpython.org/). Программы, написанные на этой библиотеке, имеют нативный для платформы интерфейс, поэтому полностью доступны для программ чтения экрана.
А выбор библиотеки [HTTPX](https://www.python-httpx.org/) обусловлен тем, что автор программы не смог заставить программу получать данные с сервера Modeus с помощью библиотеки requests или AIOHTTP.
## 1.1 Назначение и цели создания приложения
Программа предназначена для просмотра расписания занятий в Северном (Арктическом) федеральном университете. Целью создания программы является упрощение доступа к информации о расписании занятий для всех пользователей.
## 1.2 Требования к программе
- ( ) Программа должна быть кроссплатформенной и работать на Windows, Linux и MacOS. (частично выполнено)
- (x) Программа должна быть доступна для программ чтения с экрана. (На MacOS и Linux пока не озвучиваются подсказки. Чтобы их услышать, нужно нажать горячую клавишу для озвучивания статусной строки.).
- (x) Программа должна получать данные с сервера Modeus и корректно их парсить и отображать.
    Примечание: JSON ответ сервера очень сложный и не всегда корректно парсится. Например, в расписании встречаются занятия, которые ведут несколько преподавателей, и привычная строка teacher превращается в список имён, что приводило старую версию программы к вылету.
- (x) Программа должна кешировать расписание на месяц, чтобы не делать запросы к серверу каждый раз.

# 2 Проектирование приложения
## 2.1 Проектирование вариантов использования приложения
В режиме работы при подключении к интернету программа должна получить актуальное расписание на месяц и его кешировать. В онлайн режиме так же доступен просмотр чужого расписания, просмотр пересечения расписаний и авторизация.
В оффлайн режиме доступен просмотр кешированного расписания, При этом только своего. Просмотр чужого расписания и авторизация недоступны.
## 2.2 Описание выделенных классов
- schedule.Schedule - главный класс, который отвечает за получение, кеширование и расписания, поиск студентов и преподавателей, авторизацию...
- parsers.people.Person - класс, который отвечает за хранение информации о студенте или преподавателе. Список людей хранится в parsers.people.People.
- parsers.events.Event - класс, который отвечает за хранение информации о занятии. Список занятий хранится в parsers.events.Events.
    классы People и Events- это контейнеры для хранения людей/занятий. Они предоставляют методы для фильтрации, вывода, поиска, хранения и других операций над списками людей/занятий.
- parsers.people.(Student | Employee) - классы, которые наследуются от Person и отвечают за хранение информации о студенте и преподавателе соответственно.
- parsers.people.NoOne - класс, который наследуется от Person и используется для замены несуществующего студента или преподавателя.
- parsers.mess- модуль-помощник для работы с данными с сервера.
- guinput - модуль, который отвечает за ввод данных от пользователя. В нём есть классы для ввода логина и пароля, текста, выбора из списка, показа сообщения и показа QR-кода.
ical_share - модуль, который отвечает за загрузку и сохранение календаря в формате iCal на сервере, чтобы перевести его в ссылку для загрузки. В нём есть несколько функций для загрузки календаря и получения ссылки. Так же есть класс GotICalCheck(threading.Thread), который проверяет, получил ли пользователь календарь и сигнализирует об этом, чтобы закрыть окно с QR-кодом после сканирования.
- app_logic.App - класс, который отвечает за логику работы программы. Наследуется от threading.Thread и работает с классом schedule.Schedule.
main.pyw - главный файл программы. Запускает графический интерфейс и создаёт объект класса app_logic.App.

## 2.3 Проектирование пользовательского приложения
Главное окно программы состоит из двух колонок. В левой колонке производится выбор даты, в правой колонке отображается расписание на выбранную дату. В верхней части окна есть строка меню с дополнительными функциями. В нижней части окна есть статусная строка, в которой выводятся подсказки и информация о текущем состоянии программы.
При запуске программы открывается окно авторизации с полями ввода логина и пароля. Затем программа попросит ввести своё ФИО и сохранит его в кеше. Если нашлось более одного человека с таким ФИО, программа попросит выбрать нужного.
Если в главном окне сфокусироваться на определённом занятии и нажать меню файл/информация о паре, то откроется окно с подробной информацией о занятии и списком студентов, которые записаны на это занятие.
При выборе месяца в дереве дат, программа покажет расписание на этот месяц. При выборе дня в дереве дат, программа покажет расписание на этот день.
Пункт меню файл/сохранить в файл позволяет сохранить расписание в текстовый файл. При выборе программа откроет нативное окно сохранения файла.
Пункт меню файл/экспорт в календарь позволяет сохранить расписание в формате iCal. При выборе программа откроет окно с QR-кодом, который нужно отсканировать, чтобы получить ссылку на скачивание календаря. Окно с QR-кодом закроется после того, как пользователь отсканирует его.
Пункты меню файл/(чужое расписание \ пересечение расписаний) позволяют просмотреть расписание другого человека или пересечение расписаний с другим человеком. При выборе программа попросит ввести ФИО человека и покажет его расписание или пересечение расписаний.
пункт меню файл/очистить кеш позволяет удалить кеш программы. При выборе программа покажет сообщение о том, что кеш удалён, и закроется.
Пункт меню файл/выход позволяет выйти из программы. При выборе программа отправит сигнал потоку с логикой, чтобы он завершился, 1 секунду подождёт его завершения и закроется.
Пункт меню помощь/о программе открывает readme.txt в диалоговом окне.

## 3.2 Выбор и обоснование используемых библиотек
- [WXPython](https://wxpython.org/) - библиотека для создания графического интерфейса. Выбрана из-за того, что она имеет нативный для платформы интерфейс и полностью доступна для программ чтения с экрана для незрячих пользователей.
- [HTTPX](https://www.python-httpx.org/) - библиотека для работы с сетью. Выбрана из-за того, что автор программы не смог заставить программу получать данные с сервера Modeus с помощью библиотеки requests или AIOHTTP.
- [icalendar](https://icalendar.readthedocs.io/en/latest/) - библиотека для работы с календарями в формате iCal. Выбор обусловлен тем, что она позволяет легко создавать и сохранять календари в формате iCal.
- [qrcode](https://pypi.org/project/qrcode/) - библиотека для создания QR-кодов. Самая популярная библиотека для создания QR-кодов на Python. Для обработки изображений используется библиотека [Pillow](https://python-pillow.org/).
- [python-dateutil
](https://dateutil.readthedocs.io/en/stable/) - библиотека для работы с датами. Выбрана из-за того, что она позволяет легко работать с датами и временем.
- [pytz](https://pytz.sourceforge.io/) - библиотека для работы с часовыми поясами. Необходима для работы с датами и временем в московском часовом поясе.
- [pydotdict](https://pypi.org/project/pydotdict/) - библиотека для превращения словарей в своеобразные объекты, у которых можно обращаться к ключам как к атрибутам. Необходима для удобной работы с данными с сервера Modeus. Выбрана экспериментально.
- Для тестирования используется библиотека [pytest](https://docs.pytest.org/en/6.2.x/). Выбрана из-за того, что она позволяет легко писать тесты и имеет множество плагинов для тестирования.
- для компиляции программы в exe используется библиотека [pyinstaller](https://www.pyinstaller.org/). Самая популярная библиотека для компиляции программ на Python в exe.
- для создания инсталлятора используется библиотека [Inno Setup](https://www.jrsoftware.org/isinfo.php). С помощью неё можно создать самые доступные для незрячих инсталляторы.

## 3.3 Описание разработанных алгоритмов
Для получения данных через API сервера Modeus сперва программа передаёт логин и пароль и получает токен авторизации, который живёт 12 часов. С помощью этого токена осуществляется вся дальнейшая работа с сервером. При каждом запросе программа проверяет, не истёк ли токен, и если истёк, то получает новый. Токен хранится в json файле, чтобы не получать его при перезапуске программы, так как операция получения токена занимает много времени и делает несколько запросов к серверу.
